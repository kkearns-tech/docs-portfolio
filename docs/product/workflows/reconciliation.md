# End-of-Day Reconciliation

## Goal
Validate event performance and capture what is needed to restock efficiently.

## Step 1: Sales totals
- [ ] Record total units sold (by flavor if available)
- [ ] Record gross revenue estimate
- [ ] Record payment breakdown (cash/card/other) if tracked

## Step 2: Inventory consumption
- [ ] Compare starting vs ending inventory (key items)
- [ ] Note major variances (waste, loss, freebies)

## Step 3: Profitability inputs (v0)
- [ ] Record booth fees and notable expenses
- [ ] Record key operational notes (traffic, weather, staffing)

## Step 4: Restock actions
- [ ] Generate restock list from items below par
- [ ] Identify items at risk for next event

## Outputs
- Reconciled event summary (sales + notes)
- Restock list
- Exceptions logged for follow-up
